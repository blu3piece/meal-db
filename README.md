## Git Conventions

본 프로젝트에 기여할 때 원칙을 정하여 일관성 있게 커밋하기 위해 `Udacity`의 커밋 컨벤션을 따릅니다.

```
type : subject
```

### Commit Type
- feat : 새로운 기능 추가
- fix : **버그** 수정
- docs : **문서** 수정
- style : 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우
- refactor : 코드 리펙토링
- test : 테스트 코드
- chore : `빌드` 업무 수정, 패키지 매니저 수정

### Subject

서브젝트 항목은 과거형을 쓰지 않고 대문자로 시작하여 작성하도록 합니다. 또한 점도 붙이지 않도록 했습니다.
